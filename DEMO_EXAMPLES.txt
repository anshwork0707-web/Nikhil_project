‚ùì ANTICIPATED QUESTIONS & ANSWERS
================================================================================

Q1: "Why regex instead of advanced NLP like spaCy?"
A1: "For a first-semester project, regex is lightweight, fast, and educational.
     It works well for common patterns. spaCy integration is planned for v2.0."


Q2: "What if I enter an invalid date?"
A2: "The parser handles common formats. If unrecognized, deadline is set to NULL
     and the task is still saved. We could add validation in future versions."


Q3: "Can it handle recurring tasks?"
A3: "Not currently - it's designed for one-time tasks. Recurring tasks would be
     a great enhancement with a 'repeat' flag in the database."


Q4: "How does it prioritize tasks?"
A4: "Currently by deadline only - earliest deadline gets scheduled first.
     Future versions could add priority levels (high/medium/low) or difficulty."


Q5: "Is this scalable for many users?"
A5: "Current design is single-user with SQLite. For multi-user, we'd add:
     - User authentication
     - PostgreSQL database
     - Session management
     - Cloud deployment"


Q6: "What about task dependencies?"
A6: "Good question! Current version treats tasks independently. We could add
     a 'depends_on' field to create task chains in future iterations."


Q7: "Can I export the schedule?"
A7: "Not yet, but it's high on the roadmap. We plan to add:
     - PDF export
     - iCalendar (.ics) format
     - Google Calendar integration"


Q8: "What if tasks take longer than estimated?"
A8: "Users can adjust estimates manually. With usage data, we could implement
     ML to learn average durations per subject and auto-adjust predictions."
